润总您好在看到

2019-09-01 #刘润
(提问)lelly : 润总，您好！在看到每个人的商学院基础管理篇中，关于工资怎么定这一章节时，讲到了一个例子，公司通过高薪聘请了一群新员工，但又伤害到了老员工，发现周边公司这种现象很多，请问润总能否再进一步深入的支点招，先拜谢了。2019-09-01回答：1）首先，先确定自己的心态是什么。即，你认为在干同样的工作，老员工工资低，新员工工资高，是不是一个合理的现象。 如果你认为不合理，那么下面答案就是如何解决这个不合理的现象。 如果你认为合理，但是你不想让其他老员工知道新员工比他们的工资高，这件事情的难度非常巨大。 因为员工一定会在私下沟通，即便你严令禁止员工之间彼此沟通薪资体系，或者采用工资相同，用奖金等形式额外增补，达到“隐性提升”，模糊新老员工的薪资差异，也通常无济于事。 2）如果你跟我的认知一样，觉得这是不合理的。 那下面的问题就变成了如何让老员工的工资水平尽快保持一致。 　　 但是保持一致，人力资源成本就会上升，只不过这是必须要承受的成本罢了。 因为，如果不接受成本，老员工新员工之间的沟通又不可避免。 一旦沟通完之后，老员工意识到问题，他就很可能会来找你，如果你不同意涨薪，他也许就会悄悄在外面找工作。 因为这个时候在外面去找到一个类似工作，因为整体薪资水平的环境提升，你大概率会损失一位老员工。 这时，你再去招新人，你就发现你再招新人也会是涨薪后的这个价格。 而你损失的是老员工流失的时间成本，新的招聘培训成本和各项隐性支出成本。 所以我会建议，如果你能够从心态上认可，这不是一个合理现象，那就应该每个人的收入必须跟他的努力相关，而不是觉得这是一种“合理”常见的普遍现象。 只不过老员工的薪资不一定通过涨工资的手法来直接体现，而是提升绩效浮动薪资，让最终总体工资相差不大。 人才团队很重要，一定要坚守“同工同酬”这个基本理念，深深记挂在心里面。点赞数:50评论区：树 : 把掌声给苦劳，把奖金给功劳，功劳一样就给一样的钱。 卢恩英语(记单词)谢* : 同工同酬，绩效奖励给创造价值更大的同事 孙文亮 : 前面提到，用奖金形式额外增补是无济于事，后面提到可以提升绩效浮动薪资。同样是额外浮动，这里的差异是什么？「奖金」是主观行动，「绩效」是客观评价吗？ 焦浩 : 凡事解决方案都有底层逻辑。这是润总教会我最核心的事情。

明白 : 《产品名字，只有一个目的：降低用户认知成本》1 我之前写过一篇文章：文章开头，介绍了一个叫「小丁当」的工具，一个朋友看到后说，这个工具明明是 cPanel，叫什么小丁当？嗯，他说的对，这个软件是叫 cPanel。我一开始写的也是 cPanel，是这么介绍的 ——----------------cPanel，它是世界上功能强大，容易使用，因而比较受用户欢迎的虚拟主机控制系统。cPanel 是一套在网页寄存业中最享负盛名的商业软件，是基于于 Linux 和 BSD 系统及以 PHP 开发且性质为闭源软件；提供了足够强大和相当完整的主机管理功能，诸如：Webmail 及多种电邮协议、网页化 FTP 管理、SSH 连线、数据库管理系统、DNS 管理等远端网页式主机管理软件功能。--------------文章写完后，被张佳老师看到，他说 “我跟你讲，小白看到cPanel这个单词，直接就关掉页面去看下一篇教程了…”我仔细一想，还真是。于是我就修改文章，将 cPanel 取名「小丁当」，然后用「人话」解释了一下功能，大家有兴趣可以看看我重写的部分。为什么改成「小丁当」？因为它是机器猫，他是的口袋里有万能的工具。我希望用户看到这个词，就知道这篇文章会分享的是一个黑科技，一个能提供很多功能的黑科技，这样他才会读下去。举这个例子，是要分享一个概念——「用户认知成本」。曹大的文章以前也多次说过这个：。建议做互联网的人，多看几遍，隔一段时间再看。因为这个不仅仅是创业做产品需要，如果多思考一下，你会发现，日常工作沟通，写文章，谈恋爱等等，只要是「和人沟通」的事情。都和用户认知有关系。（1）什么是用户认知？就是用户看到这个产品，会想到什么？比如：看到 cPanel，和下面的一段话，就会想到计算机。看到小丁当，就会想到机器猫，一个万能的工具。（2）什么是用户认知成本？就是用户从看到这个产品，到发生兴趣，所需要花费的成本（时间、精力、知识、思考、动作（搜索、询问、请教）等等）（3）什么是降低用户认知成本？就是要让用户从看到你的产品，到发生兴趣之间，所需要花费的成本尽可能最小。（4）怎么降低用户认知成本？我们做产品，无论是APP，小程序、海报、文章等等，第一个进入用户眼睛的，是名字。如何降低用户对于名字的认知成本？一句话：把名字和熟悉的事物联系起来。《超级符号原理》，讲的就是这个，这并不是新理论，我们每个人都用过。它另一个名字叫类比，或者比喻。把 A 比作 B。A 叫本体B 叫喻体。这个大家都熟悉。超级符号，是一个非常强大的喻体，强大到你看了就会觉得很熟悉，很信任，很想买。cPanel 叫做小丁当，就是把 cPanel 和 小丁当这个熟悉的事物联系起来，降低用户认知成本。一个小程序统计平台，叫阿拉丁，就是把平台和阿拉丁神灯联系起来，比喻什么都能做的意思。知识星球，为什么叫知识星球？把知识社群、和星球联系起来，用户就知道，这个产品里面有好多社群，因为宇宙有很多星球。现在你知道了，做产品，需要找到一个熟悉的事物，把产品和它联系起来。-----------------------------------我们再来看一个真实案例。2
